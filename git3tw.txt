🛡️ 메인에 영향 안 주고 작업하는 3단계 공식
1단계: 작업 시작 전, 무조건 브랜치부터 새로 만들고 시작하기
VS Code를 켜자마자 main에서 작업을 시작하면 안 됩니다. 가장 먼저 할 일은 안전한 실험실(새 브랜치)을 만드는 것입니다.

Bash
# 1. 안전한 원본 기지인 main으로 이동합니다.
git switch main

# 2. 메인에서 뻗어나온 완전히 새로운 비밀 실험실을 만듭니다. (예: feature-abc)
git switch -c feature-abc

2단계: 작업하는 동안 "자주" 커밋(Commit)하기 (가장 중요 ⭐)
코드를 작성하고, 파일을 추가하는 동안 '장바구니(add)'에만 담아두고 자리를 뜨면 안 됩니다. 아까처럼 main으로 이동할 때 따라가 버리기 때문이죠.
어느 정도 의미 있는 작업이 끝날 때마다 꼭 커밋까지 완료해서 도장을 찍어두어야 합니다.

Bash
# 1. 수정한 파일들을 장바구니에 담기
git add .

# 2. 이 브랜치에 안전하게 영구 저장(박제)하기
git commit -m "feat: 로그인 화면 기능 구현 완료"

3단계: 작업이 완벽하게 끝났을 때만 메인에 합치기 (Merge)
실험실 브랜치에서 코딩도 다 하고, 테스트도 완벽하게 끝내서 이제 진짜 원본 기지에 적용해도 되겠다는 확신이 들 때! 그때만 main으로 가져와 합칩니다.

Bash
# 1. 메인 브랜치로 이동 (여기서 내 연구실 파일들이 슥 사라지는 마법을 감상합니다)
git switch main

# 2. 완벽하게 검증된 내 연구실의 성과를 합칩니다.
git merge feature-abc
